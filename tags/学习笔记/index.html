Iterable(可迭代类)

Iterable^1是一个Collection，是一个抽象类，表示一组可以依次访问(access sequentially)的对象。

Iterable是一个Collection

Collection是一个对象，它表示一组对象（称之为元素）。
Iterable是一种Collection。

常见的三种Collection

  1. List
  2. Set,（不重复）
  3. Map（ Key-Value）

Iterable是一个抽象类

换言之，我们不能实例化Iterable，但是我们可以借由List和Set实现对Iterable的实例化。
List和Set派生自Iterable，而Map则不然。但是我们可以通过正确的使用Map的values（entries）从而以Iterable的方式读取Map。

Iterable

与List不同，用index访问Iterable并不保证是高效的。

Iterable原生没有“[ ]”操作符，可以使用elementAt()。

访问Iterable

1. 使用for-in循环
2. 使用first 和last

类似C++，first是第一个元素，last是最后的元素。
注意：last需要访问前面的所有元素，所以在较大的iterable时，last可能是慢的。

3. 使用firstWhere

firstWhere的参数是一个函数。
注意：
=>（箭头符）相当于return

检查元素

any 和 every

any： iterable中有一个元素符合条件
every： iterable中所有元素符合条件

什么是RPC框架？

RPC框架，即Remote Procedure Call（远程过程调用），是一个计算机通信协议。
该协议允许计算机远程调用另一台计算机上的程序，实现Server/Client模式。

什么是gRPC框架？

gRPC框架的官方文档：https://grpc.io/
gRPC框架是Google开发的一款开源的、高性能的、通用的面向HTTP/2的RPC框架。

为什么是通用的？

gRPC可以实现多语言端的通信。包括但不限于Golang, Python, Dart等。

gRPC框架使用protocol buffers作为其IDL语言。
IDL: Interface Description Language ,接口定义语言。

简单说来，通过protocol buffers描述定义各种接口和各种数据结构，再通过编译器（Protoc）自动化实现各个语言版本。
当然这种实现仅仅是提供一个框架，而要实现其功能，还需要实际上填充代码。
